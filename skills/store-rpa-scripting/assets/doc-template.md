# 过程文档模板

Phase 4 生成的过程文档应遵循以下模板。文件命名与脚本对应：`rpa_[task_name]_doc.md`。

## 模板

```markdown
# [任务名称] — RPA 自动化过程文档

## 1. 概述
- **目标**: [一句话描述自动化目标]
- **范围**: [入口 URL、会话类型、覆盖范围]
- **生成日期**: [YYYY-MM-DD]

## 2. 环境准备

### 2.1 探索环境（CLI，Phase 1）
- 已安装：`uv tool install ziniao`（或等价），`ziniao config show` 可正常输出
- 探索命令见过程文档第 3 节记录

### 2.2 独立脚本依赖（Phase 3）
```bash
pip install ziniao
# ziniao 包内含 ziniao_webdriver（紫鸟 HTTP 客户端）并声明了 nodriver 依赖，
# 一条命令即可；不必克隆 ziniao 源码仓库，也不需要 CLI 守护进程常驻。
```

### 2.3 紫鸟客户端（脚本内 ZiniaoClient）
- 客户端路径: `D:\ziniao\ziniao.exe`（按实际修改）
- 确保客户端版本支持 WebDriver 模式

### 2.4 账号配置
在脚本中修改 `ZINIAO_CONFIG`：
| 字段 | 说明 | 示例 |
|------|------|------|
| client_path | 紫鸟客户端路径 | `D:\ziniao\ziniao.exe` |
| company | 企业名 | `mycompany` |
| username | 登录用户名 | `admin` |
| password | 登录密码 | `***` |

### 2.5 输入数据（如需）
- 文件路径: `tasks.csv`
- 格式说明: [列名和含义]
- 示例行: [1-2 行样例数据]

## 3. 探索记录

### 3.1 页面分析
| 页面 | URL | 关键元素 | 备注 |
|------|-----|----------|------|
| [页面名] | [URL] | [选择器列表] | [特殊说明] |

### 3.2 操作流程验证
[Phase 1 中每一步探索的记录，包括：]
- 执行的终端命令（`ziniao ...`）及关键参数
- 终端输出或 `--json` 解析结果摘要
- 发现的选择器（注明是否来自 `snapshot --interactive` 的 Selector 列）
- 遇到的异常及处理方式

### 3.3 API 接口（如有）
| 接口 | Method | URL | 用途 |
|------|--------|-----|------|

| # | 操作 | 选择器/参数 | 等待条件 | 预期结果 |
|---|------|------------|----------|----------|
| 1 | ... | ... | ... | ... |

## 5. 脚本使用

### 5.1 运行方式
```bash
python rpa_[task_name].py
```

### 5.2 配置项说明
[脚本中需要用户修改的配置项列表]

### 5.3 输出说明
- 日志: 控制台输出运行进度
- 截图: `screenshots/` 目录下的关键步骤截图
- 报告: 执行完成后的成功/失败汇总

## 6. 异常处理

| 异常 | 可能原因 | 解决方式 |
|------|----------|----------|
| 客户端启动失败 | 路径错误/权限不足 | 检查 client_path，以管理员运行 |
| 店铺打开失败 | store_id 错误/代理过期 | 核实 ID，检查代理状态 |
| 元素未找到 | 页面改版/加载慢 | 更新选择器，增大 timeout |
| 操作超时 | 网络慢/页面卡顿 | 增加重试次数和间隔 |

## 7. 维护说明
- 页面改版时需重新执行 Phase 1 探索，更新选择器
- 定期检查脚本运行日志，关注失败率变化
- 紫鸟客户端更新后需验证 API 兼容性
```

## 生成要点

- 将 Phase 1 探索过程中的**实际 `ziniao` 命令与输出摘要**记录到"探索记录"章节
- 将 Phase 2 确认的步骤表格完整复制到"操作步骤详解"章节
- 脚本文件名和文档文件名保持对应（如 `rpa_batch_price.py` + `rpa_batch_price_doc.md`）
- 文档应足够详细，让不了解项目的人也能按文档复现整个流程
